HTML 5 (HyperText Markup Language)

Hai perkenalkan di adalah HTML 5, dari kepanjangan dari HTML, kita tau bahwa HTML merupakan bahasa Markup (Markup Language). berdasarkan arti kata Mark pada Markup yang artinya Menandai begitu juga dengan syntaxnya(gaya penulisan kode) menandai di sini adalah setiap konten akan di tandai/dibungkus.

Perhatikan kode di atas!, itu merupakan sebuah format text html paling sederhana. untuk syntaxnya tidak usah pusing dulu, kita akan membahasnya satu persatu. terus mana yang di bilang Ditandai(Markup)? coba perhatikan pada text berwarna putih bertuliskan kitaBisaKoding ada dua buah tulisan yang membungkusnya. title dan /title kedua text tersebut dinamakan TAG / ELEMENT. dan Element title terdapat Element yang membungkusnya yaitu element head dan /head.

Lah kenapa harus terdapat 2 element yang sama? karena terdapat element pembuka dan element penutup. Element penutup ditandai dengan slash ( / ) sebelum nama element nya.
setiap element memiliki fungsi nya masing-masing bukan hanya sekedar membungkus tetapi dapat memformat text, menampilkan gambar. contohnya element img berfungsi untuk menampilkan gambar ke layar.

Nah pada kasus ini element img tidak di tutup dengan /img kenapa? karena element img tidak mempunyai content di dalamnya. Element seperti ini dinamakan Singleton Tag

terus apa arti kata HyperText pada HTML? HyperText adalah text yang dapat menghubungkan dan mengakses sebuah dokumen ke dokumen yang lain. Dengan kata lain, dokumen HTML kita dapat saling terhubung ke dokumen HTML kita yang lain melalui HyperText.
HyperText yang dimaksud dapat berupa anchor / link

Tapi kenapa ada angka 5 nya?

angka 5 pada HTML 5 adalah versi dari HTML iti sendiri, pada saatini HTML Berada pada versi 5
terus apa bedanya dengan versi sebelumnya? beberapa pembaharuan pada versi 5 antara lain, diantaranya pada segi multimedia dan grafis kita dapat menggunakannya secara native. Penambahan element baru seperti video, audio, canvas, suppport terhadap gambar vektor (SVG (scalable Vector Graphics)), memperbanyak semantic element seperti header, main, nav dan masih banyak lagi. dan beberapa optimisasi lainnya. Semua itu akan kita pelajari.
