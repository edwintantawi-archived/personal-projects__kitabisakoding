WEBSITE

Kehidupan sehari hari kita sudah tidak terlepas dari namanya website. Pada saat kita mencari resep masakan, mencari tutorial, mencari hiburan, bahkan berbelanja. Tapi apa itu Website? Website atau situs web adalah sekumpulan halaman web yang saling terhubung satu sama lain yang biasanya pada Paladen (server) yang sama di Internet. Dimana setiap halaman web tersebut berisikan informasi. Sebuah website dapat diakses melalui jaringan Internet maupun Jaringan lokal (LAN), setidaknya website tersebut berada pada sebuah Web server. Kita dapat mencari alamat web melaui URL (Uniform Resource Locator).

Siapa penemu website ?

Penemu website / situs web adalah Sir Timothy John "Tim" Berners-Lee. ketika ia bekerja di CERN (Conseil Européen pour la Recherche Nucléaire). Web awalnya dibuat dan dikembangkan untuk memenuhi permintaan dalam berbagi informasi secara otomatis antar ilmuwan di universitas dan lembaga di seluruh dunia. Web pertama di CERN didedikasikan sebagai web pertama di dunia.

Client-Server

yap, pada website terdapat istilah Client-Server, karena pada website terjadi interaksi antara client dan server.

Client

Client di sini berupa browser yang kita gunakan sebagai user untuk mengakses situs web. Pada Client juga terjadi Proses pada browser / disisi pengguna yaitu client akan mengirimkan request ke server (request sini adalah, client akan meminta data website yang akan di tampilkan berupa HTML/CSS/JAVASCRIPT) kepada server. bagaimana client tau bahwa itu adalah server yang dituju? ketika kita menuliskan alamat website berupa domain, (domain merupakan suatu cara penamaan IP address agar mudah di baca dan di hafal yang sebenarnya domain tersebut adalah IP address web server kita)

Server

Server disini adalah Web server yang berfungsi sebafai tempat kita meletakkan seluruh sumber/file website kita, pada server akan terjadi sebuah proses dimana, ketika client mengirimkan request kepada server, server akan memberikan respon berupa data yang di request oleh client ( contohnya, ketika client membuka www.tokopedia.com/home maka server akan mengirimkan file yang berhubungan dengan "home" ke client). ketika client menerima respon dari server, maka client dapat memproses file tersebut dan menampilkannya pada jendela browser kita.

WebServer

Web server adalah sebuah komputer server yang menyimpan data data website kita (HTML, CSS, JS, gambar, dan lain lain) agar dapat di akses oleh publik. Server harus memiliki performa yang baik untuk mengelola traffic request client dan mengelola data, karena client bisa saja lebih dari satu, bahkan ratusan maupun ribuan

URL

URL (Uniform Resource Locator) adalah sebuah format penulisan alamat suatu sumber dokumen / website / gambar.

Bagian dari URL

(contoh: https://www.facebook.com/friends)

  • https/https : Protokol komunikasi data untuk World Wide Web
  • www : www / World Wide Web merupakan suatu ruang informasi kumpulan web server di dunia
  • facebook : Nama alamat website
  • .com : Nama domain (dalam kasus ini adalah ".com" dimana berfungsi untuk commercial)
  • /setting : Nama sub direktori (folder/ruang di dalam website)
  • Jika kamu pernah melihat URL Seperti (info.namaweb.com atau store.namaweb.com) info. dan store. adalah sebuah subDomain

HTTP/HTTPS

HTTP (HyperText Transfer Protocol) dan HTTPS (HTTP Secure) berfungsi sebagai protokol permintaan-respons dalam model komputasi klien-server. Peramban web, misalnya, mungkin klien dan aplikasi yang berjalan di komputer yang meng-hosting situs web mungkin adalah server. Klien mengirimkan pesan permintaan HTTP ke server. Server, yang menyediakan sumber daya seperti file HTML dan konten lainnya, atau melakukan fungsi lain atas nama klien, mengembalikan pesan respons ke klien. Respons tersebut berisi informasi status penyelesaian tentang permintaan dan mungkin juga berisi konten yang diminta di badan pesannya.

HTTPS

HTTPS (HTTPS Secure) adalah HTTP yang di enkripsi menggunakan Transport Layer Security (TLS) atau Secure Socket Layer (SSL), dengan begitu protokol HTTPS Lebih aman ketika terjadi pertukaran data. Sangat penting digunakan ketika sebuah website berhubungan dangan banking, data data penting, informasi pribadi. Karena setiap pertukaran data, seluruh data penting akan terenkripsi.

Beberapa pesan respon yang di kembalian yang sering di jumpai

  1. 100 Lanjutkan : server telah menerima header permintaan, dan klien harus melanjutkan untuk mengirim permintaan body
  2. 101 Protokol Switching : pemohon telah meminta server untuk beralih protokol
  3. 200 OK : permintaan sukses/berhasil dilakukan
  4. 204 Tidak ada Konten : permintaan berhasil, tetapi tidak mengembalikan konten apapun
  5. 302 Ditemukan : Halaman yang diminta telah pindah sementara ke URL baru
  6. 306 Beralih Proxy : Tidak lagi digunakan
  7. 403 Forbidden : Permintaan itu permintaan hukum, tetapi server menolak untuk menanggapi itu
  8. 404 tidak ditemukan : Halaman yang diminta tidak dapat ditemukan

DNS

DNS (Domain Name System) adalah sebuah sistem penamaan IP address sebuah website.
Sebenarnya ketika ingin mengakses sebuah website kita memerlukan IP Address website tersebut, ini cukup merepotkan karena IP Address suatu website sulit di ingat (contoh: 192.168.10.8). Cukup mrepotkan bukan?.

Domain

Tanpa domain, jika user ingin mengakses suatu website maka mereka harus mencari menggunakan IP Address website tersebut. Domain berfungsi menggantikan IP Address web dengan karakter yang mudah di ingat. (contoh: facebook.com)

ada beberapa jenis tipe domain

  • .com : untuk komersial
  • .org : untuk organisasi
  • .gov : untuk lembaga pemerintahan
  • .id / .us / .sg : untuk menunjukan lokasi dari web tersebut (contoh: .id = Indonesia)
  • dan masih banyak lagi

Disnilah fungsi dari DNS, menterjemahkan domain yang kita masukan pada browser menjadi IP Address yang dimengerti oleh komputer.
